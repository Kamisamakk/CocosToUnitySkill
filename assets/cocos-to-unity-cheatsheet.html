🚀 v3.4 核心改进

  • 脚本 .meta 自动生成：translator.py 生成 .cs 时自动生成对应 .meta（包含唯一 GUID）
  • 预制体直接引用：Phase 4 转换时扫描脚本 GUID，直接写入 YAML 的 m_Script 字段
  • Phase 5 可选：由于预制体转换时已直接引用，关联步骤成功率 100%
  • 执行顺序优化：Phase 2 脚本 → Phase 3 资产，确保 .meta 在预制体转换前生成

🎯 适用场景

  • 将 Cocos Creator 2.x / 3.x 项目迁移到 Unity
  • 转换 .fire / .scene / .prefab 文件
  • 将 TypeScript / JavaScript 组件翻译为 C# MonoBehaviour
  • 迁移 Spine / DragonBones / 音频 / UI 到 Unity
  • 诊断映射不匹配问题

⚡ 技术栈

  • Python 3.9+：仅用标准库，无需 pip install
  • Unity 2020.3+：推荐 2022.3 LTS
  • 跨平台：Win / macOS / Linux
  • LLM 驱动：OpenAI / Anthropic 自动切换，无 LLM 时降级骨架生成
  • AI Agent：可被 AI Agent 调用完成完整迁移流程

常见坑点 & 对策

❌ TMP 编译错误

在安装 TextMeshPro 包之前引用了 TMP → 编译失败。
对策：默认用 legacy UI.Text，确认 TMP 已装再切换。

❌ 3D 坐标手性

Cocos 3D 右手系 (Z+前)；Unity 左手系。
对策：所有 3D 位置/旋转翻转 Z 轴。

❌ 2.x 旋转符号

Cocos 2.x 顺时针旋转；Unity 逆时针。
对策：convert_scene.py 自动取反处理。

❌ 2.x 锚点原点

2.x 默认原点左下角；3.x 为中心。
对策：审计自动检测并应用正确偏移公式。

✅ Spine 版本锁定

Skeleton 导出版本必须精确匹配运行时版本。
对策：审计从 JSON "spine" 字段检测版本。

✅ 脚本 .meta 自动生成

translator.py 生成 .cs 时自动生成对应 .meta。
对策：无需手动维护 GUID，pipeline 自动处理。

✅ 预制体直接引用

Phase 4 转换时扫描脚本 GUID，直接写入 YAML。
对策：Phase 5 成为可选步骤，成功率 100%。

✅ PPU 坐标转换

Cocos 像素值直接导入导致位置巨大。
对策：--ppu 100 自动除以 PPU。

快速开始
  1. 完整迁移：python scripts/pipeline.py --cocos <cocos-root> --unity <unity-root>
  2. 单阶段执行：python scripts/pipeline.py --cocos <src> --unity <dst> --phase 2
  3. 指定 PPU：python scripts/pipeline.py ... --ppu 100 --convert-pos
  4. 生成图集：python scripts/pipeline.py ... --atlas
  5. 验证报告：python scripts/verify_migration.py --unity <path> --report report.md

约定规范

📂 资产暂存

所有迁移内容先放 Assets/_Ported/，不覆盖已有 Unity 资产。

🔑 Manifest 为真

manifest.json 是资产身份的唯一真实来源 (Cocos UUID → Unity GUID)。

🎨 默认管线

Built-in RP + 2D Sprite。URP 需覆盖纹理导入设置和 Shader。

🤖 AI Agent

支持 OpenAI (OPENAI_API_KEY) / Anthropic (ANTHROPIC_API_KEY)，无 LLM 时降级骨架生成。
